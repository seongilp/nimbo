# 개인정보처리방침

**Nimbo는 셀프호스팅(self-hosted) 소프트웨어입니다.** 당신의 서버에서 직접 실행되며,
콘솔이 다루는 모든 데이터는 **당신의 서버에만** 저장됩니다. 개발자나 제3자에게 전송되는
정보는 없으며, 원격 분석(텔레메트리)이나 추적도 하지 않습니다.

> 앱 안에서도 같은 내용을 볼 수 있습니다: 로그인 화면 하단의 **개인정보처리방침**(`/privacy`).

최종 업데이트: **2026-07-09**

## 1. 수집·처리하는 정보

- **로그인 정보** — 서버의 OS 계정으로 인증합니다. 비밀번호는 **저장하지 않고** 서버의
  시스템 라이브러리(libcrypt)로 검증만 합니다.
- **세션** — 로그인 시 HMAC 서명 쿠키(`nimbo_session`)가 발급되며 서버에서만 검증됩니다.
- **접속 IP 주소** — 로그인 허용 대역(IP 접근제어), 감사 로그, 무차별 대입 차단(fail2ban)에
  사용되며 서버 내에만 기록됩니다.
- **감사 로그·설정** — 사용자 작업/로그인 이력과 서버 설정은 서버의 `/etc/nimbo` 및
  로그(journald)에 로컬 저장됩니다.

## 2. 이용 목적

인증, 접근 제어, 보안(무차별 대입·비정상 접근 차단), 그리고 서버 관리 기능 제공에만
사용합니다.

## 3. 제3자 제공 및 외부 전송

**없음.** Nimbo는 어떤 정보도 외부 서버로 전송하지 않습니다. 광고·분석 도구·추적 스크립트를
포함하지 않으며, 폰트 등 리소스도 앱에 번들되어 외부 CDN을 호출하지 않습니다.

## 4. 쿠키

인증용 세션 쿠키(`nimbo_session`) 하나만 사용합니다. `HttpOnly`로 설정되어 자바스크립트로
읽을 수 없으며, HTTPS 접속 시 `Secure` 플래그가 적용됩니다. 광고·추적 쿠키는 없습니다.

## 5. 보관 및 파기

모든 데이터는 당신의 서버에 있으며 당신이 직접 관리합니다. 서버에서
`sudo nimbo uninstall --purge`를 실행하면 설정·인증 정보(`/etc/nimbo`)를 포함해 완전히
삭제됩니다.

## 6. 보안 조치

Caddy를 통한 HTTPS, 앱의 루프백(127.0.0.1) 바인딩, 첫 로그인 네트워크 고정(IP 접근제어),
선택적 2단계 인증(TOTP), 로그인 잠금 및 fail2ban, 셸을 거치지 않는(argv) 권한 명령 실행
등으로 보호합니다. 자세한 내용은 [SECURITY.md](SECURITY.md)를 참고하세요.

## 7. 소개 웹사이트

공개 소개 페이지(GitHub Pages)는 정적 페이지로 추적 쿠키나 분석 도구를 사용하지 않습니다.
다만 호스팅 제공자(GitHub)의 서버 접근 로그는 GitHub의 정책을 따릅니다.

## 8. 이용자의 권리

데이터가 전적으로 당신의 서버에 있으므로, 언제든 직접 열람·수정·삭제할 수 있습니다.
Users 앱에서 접근 허용 계정·IP를, Security 앱에서 2FA·방화벽을, Audit Log 앱에서 활동
기록을 확인/관리할 수 있습니다.

## 9. 문의

보안 취약점은 공개 이슈 대신 [비공개 보안 신고](https://github.com/seongilp/nimbo/security/advisories/new)로
접수해 주세요.
